Django란?

Python 웹 프레임워크 중에 하나이며, 에이드리안 홀로바티와 사이먼 윌리슨이 python을 이용해 어플리케이션을 만들기 시작하면서 처음 개발되었다.
django는 흔히 아는 디자인 패턴인 MVC(model, view, controller) 패턴과 같은 MVT(model, view ,template) 패턴으로 되어있다.
MVC 패턴의 V가 MVT 패턴의 T에 해당하고, MVC 패턴의 C가 MVT 패턴의 V에 해당한다.

Django 동작 원리

  1. 클라이언트로부터 요청을 받으면 URLconf 모듈을 이용하여 URL을 분석한다.
  2. URL 분석한 결과를 통해 해당 URL과 매칭되는 view를 실행한다.
  3. view는 해당하는 로직을 실행하고 데이터베이스 처리가 필요하다면 model을 통해 처리하고 결과를 반환 받는다.
  4. view의 해당하는 로직 처리가 끝나면 template을 사용하여 클라이언트에 전송할 HTML 파일을 생성한다.
  5. view는 최종 결과로 HTML 파일을 클라이언트에게 보내서 응답한다.

Django 장점

  • django 프로젝트는 app 단위로 구성이 되고 app은 model, view, template으로 구성이 되어 있어서 분업하기에 좋다.
  • 관리자 페이지를 기본적으로 제공한다.(phpMyAdmin과 같은 데이터베이스를 컨트롤할 수 있는 페이지)
  • 보안에 강하다.(XSS, SQL injection, Clickjacking...)
  • 이 외에도 웹사이트에 필요한 기능들이 많이 내장되어 있다.

Django 단점

  • 기본적으로 제공되는 기능에 대한 최적화가 어렵다.
  • Python은 인터프리터 언어이기 때문에 속도적인 부분에서 느리다. 게다가 Django에는 내장되어 있는 기능들이 많기 때문에 무거워서 다른 프레임워크보다 느리다.
